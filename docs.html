Authentication

üîê User Authentication (JWT)

Get started by registering and logging in to obtain a JWT token for accessing user-specific endpoints.

1. Register a New Account

POST /api/v1/auth/register
{ "email": "user@example.com", "password": "securePassword123", "firstName": "John", "lastName": "Doe" }

2. Login to Get JWT Token

POST /api/v1/auth/login
{ "email": "user@example.com", "password": "securePassword123" }

Response:

{ "success": true, "message": "Login successful", "data": { "user": { "id": "user_1234567890", "email": "user@example.com", "firstName": "John", "lastName": "Doe" }, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "refreshToken": "refresh_token_here" } }

3. Use JWT Token

Include the JWT token in the Authorization header for all authenticated requests:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

üîë API Key Authentication

After creating an organization and API keys, use them to access OnlyFans data endpoints.

1. Create Organization

First, create an organization (requires JWT authentication):

POST /api/v1/organizations
{ "name": "My Content Organization", "description": "Managing multiple OnlyFans creators", "logo": "https://example.com/logo.png" }

2. Create API Key

Create an API key for your organization:

POST /api/v1/api-keys
{ "name": "Production API Key", "organizationId": "org_1234567890", "permissions": {"read": true, "write": false}, "rateLimit": 1000 }

Save your API key securely! The full key is only shown once during creation.

3. Use API Key for Data Access

Use the API key in the X-API-Key header to access OnlyFans data:

X-API-Key: of_prod_1234567890abcdef1234567890abcdef

Example Request:

GET /api/v1/onlyfans/organizations/{orgId}/models
// Headers: // Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... // X-API-Key: of_prod_1234567890abcdef1234567890abcdef

üîí Security Best Practices

  • Never expose API keys in client-side code or public repositories
  • Use environment variables to store API keys securely
  • Rotate API keys regularly and revoke unused ones immediately
  • Monitor API key usage through the dashboard
  • Use the minimum required permissions for each API key
